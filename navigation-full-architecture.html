Option 2: Per-Section Controls

Each document type has its own mini view controls.

‚úì Pros

  • Tailored controls per type
  • More flexible
  • Meetings can have special handling
  • Each section remembers its view

‚úó Cons

  • More complex interface
  • Takes more space
  • Potentially overwhelming

Key Design Decisions

Meeting Documents Special Case

Meeting-related documents (minutes, notes, agendas, transcripts) are naturally grouped by meeting date. They could:

  • Always show grouped by meeting (no flat list option)
  • Include meeting type indicators (Regular, Special, Work Session)
  • Show document completeness (which docs are available for each meeting)

Search Behavior

  • Global search searches across all document types
  • Results could show grouped by type or mixed
  • Search within a meeting finds content in any of its documents

Hybrid Approach?

We could also do a hybrid:

  • Global view toggle (List/Grouped)
  • But "Grouped" means different things for different sections:
    • Ordinances ‚Üí By decade
    • Resolutions ‚Üí By year
    • Interpretations ‚Üí By code section
    • Meetings ‚Üí Always by meeting date

My Recommendation

Option 1 with smart defaults: Global controls, but the "Group" view intelligently groups each document type by what makes most sense for that type. Keep it simple but smart.
