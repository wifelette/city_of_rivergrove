Should we add a Tooltips section to the Style Guide?

Yes, I think we should! Here's why:

  • Consistency: Having documented standards ensures all tooltips look and behave the same
  • Reusability: Developers can easily apply the pattern to new features
  • Accessibility: We can document ARIA attributes and keyboard considerations
  • Not overkill: Tooltips are a common UI pattern we use in multiple places

Suggested Style Guide Section:

## Tooltips

Tooltips provide contextual information on hover. Used for:
- Form fields (explaining blank/filled status)
- Signatures (indicating presence in original)
- Help text for complex features

### Standard Implementation
- Position: Above element (bottom: 100%)
- Background: #333
- Text: white, 0.75rem
- Padding: 5px 10px
- Appears on hover with 0.2s transition

### Accessibility
- Use `aria-label` or `title` as fallback
- Ensure contrast ratio meets WCAG standards
- Consider touch devices (tooltips don't work well)

### CSS Utility Class
`.tooltip` - Apply to any element needing a tooltip
Use `data-tooltip="text"` for dynamic content
