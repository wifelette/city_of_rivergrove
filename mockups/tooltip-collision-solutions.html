Features:

  • Quick transition (0.15s) for responsiveness
  • Optional 0.3s delay for dense areas only
  • Subtle shadow for better visibility

üéØ Recommended Approach

Two-tier system:

  1. Default tooltip (.tooltip)
    • No delay for good UX
    • High z-index (999999) when active
    • Slightly more margin for breathing room
  2. Dense area tooltip (.tooltip.dense-area)
    • Small 0.3s delay (not 0.5s like before)
    • Only for signature blocks and dense forms
    • Same styling otherwise

Implementation:

/* Base tooltip */
.tooltip::after {
    /* ... standard styles ... */
    z-index: 10000;
    transition: opacity 0.15s ease-in-out;
}

.tooltip:hover::after {
    opacity: 1;
    z-index: 999999; /* Ensure active tooltip wins */
}

/* For dense areas only */
.tooltip.dense-area::after {
    transition: opacity 0.15s ease-in-out 0.3s;
}

When to use .dense-area:

  • Signature blocks with multiple signatures close together
  • Forms with many adjacent filled fields
  • Tables with tooltips on every cell

This gives us the best of both worlds: Responsive tooltips in most cases, with collision mitigation where needed.
