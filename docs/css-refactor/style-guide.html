üìù Enhanced Lists

Special list formatting for legal documents with parenthetical markers. All list markers display in blue (#0969da) without bullets.

üìñ View Complete Lists Style Guide

Alpha Parenthetical Lists (a, b, c)
  • (a) The City Council shall meet at least once per month
  • (b) Special meetings may be called by the Mayor or any three Council members
  • (c) All meetings shall comply with Oregon Public Meeting Law

CSS Class: .alpha-list | Marker: .list-marker-alpha

Numeric Parenthetical Lists (1, 2, 3)
  • (1) First numbered requirement or condition
  • (2) Second numbered requirement or condition
  • (3) Third numbered requirement or condition

CSS Class: .numeric-list | Marker: .list-marker-numeric

Roman Numeral Lists (i, ii, iii)
  • (i) Initial review by Planning Commission
  • (ii) Public hearing and comment period
  • (iii) Final determination by City Council

CSS Class: .roman-list | Marker: .list-marker-roman

Nested Lists
  • (a) Parent alpha item with nested content
    • (1) First nested numeric item
    • (2) Second nested numeric item
  • (b) Second parent alpha item
    • (1) Another nested item
    • (2) Another nested item

Nesting preserves visual hierarchy through indentation

  • Preprocessor: standardize-list-format.py normalizes certain patterns

  • Marker color: All markers display in blue (#0969da)

üìö Style Guide Conventions

Special formatting used in this style guide for clarity and emphasis. These styles are for documentation purposes only and don't appear in the production site.

Callout Boxes

We use colored callout boxes to highlight different types of information:

Quick guide:
Green = "Here's something helpful!"
Red = "Don't do this!"
Blue = "This is a core principle"
Yellow = "Be careful here"
Info Box (Green): Used for helpful supplementary information, tips, common usage patterns, and "good to know" context. These are positive, actionable insights that enhance understanding.
Warning Box (Red): Used for critical warnings, important restrictions, things that must NOT be done, and potential issues to avoid. These require immediate attention to prevent problems.
Principle Box (Blue): Used for foundational rules, key design principles, and architectural decisions that guide the entire implementation. These are the "why" behind our choices.
Note Box (Yellow): Used for important caveats, exceptions, edge cases, and "watch out for" situations that aren't quite warnings but need attention. Think of these as "proceed with caution" notices.
Color Code Display

Hex color codes are automatically enhanced with visual swatches throughout the documentation:

When you see a color like #0969da or #059669 or #dc2626 in the text, it automatically displays with a small color preview. This makes it easy to see what color is being referenced without having to look it up.

This feature works for:

  • 6-digit hex codes: #0969da
  • 3-digit hex codes: #fff
  • Any valid hex color mentioned in documentation
Code Snippets

Different types of code are displayed with appropriate formatting:

Dark theme code blocks: Used for CSS, JavaScript, and HTML code
/* CSS Example */ .my-class { color: var(--color-primary); padding: var(--space-padding-md); }
Inline code: Used for short references like class="example" or #0969da
üí° Copy to Clipboard: Hover over any code block to reveal a "Copy" button in the top-right corner. Click to instantly copy the code to your clipboard!
Visual Indicators
  • NEW - Indicates recently added features or sections
  • Bold text - Used for emphasis and important terms
  • Italic text - Used for subtle emphasis or examples
  • üìä Emojis - Used as visual markers for different sections and concepts
Usage Guidelines
These conventions are designed to make the documentation clearer and easier to scan. Use them consistently to maintain a professional and helpful style guide.
