Basic Tables

All tables in City documents follow a consistent format with bold headers and proper alignment.

Standard Table Structure
Column Header 1 Column Header 2 Column Header 3
Row 1, Cell 1 Row 1, Cell 2 Row 1, Cell 3
Row 2, Cell 1 Row 2, Cell 2 Row 2, Cell 3
| **Column Header 1** | **Column Header 2** | **Column Header 3** | |---------------------|---------------------|---------------------| | Row 1, Cell 1 | Row 1, Cell 2 | Row 1, Cell 3 | | Row 2, Cell 1 | Row 2, Cell 2 | Row 2, Cell 3 |

Key Requirements

  • Always use bold for column headers: **Header Text**
  • Use at least 3 dashes for separator row
  • Align columns with pipes for readability
  • Maintain consistent column widths

Best Practices for Financial Tables

  • Use green color (#22c55e) for positive changes
  • Use red color (#ef4444) for negative changes
  • Bold the totals row
  • Add a double or thick border above totals
  • Consider subtle background for totals row

Best Practices

‚úÖ Do's

  • Always bold column headers using **Header Text**
  • Use consistent column alignment throughout the table
  • Escape special characters in markdown tables (\*, \|)
  • Include units in headers when applicable (e.g., "Fee (USD)")
  • Use footnotes for explanatory text rather than cramming into cells
  • Bold totals rows for emphasis
  • Use color coding consistently (green for positive, red for negative)
  • Test tables at different screen sizes for responsive behavior

‚ùå Don'ts

  • Don't use HTML tables unless absolutely necessary
  • Don't include lengthy explanations in cells (use footnotes instead)
  • Don't mix alignment styles within a table
  • Don't forget to escape asterisks when using footnote markers
  • Don't use inline styles unless necessary for special formatting
  • Don't create tables wider than the content area
